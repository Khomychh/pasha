# Client-side conversation state

The backend needs to hold conversation context so the Persona's replies stay coherent across a Guest's messages. We considered keeping this in server-side in-memory state keyed by session cookie, but that state is lost on every container restart/redeploy (Docker on a single Hetzner instance), silently dropping active conversations. Instead, the browser holds the full message history and resends it with each request; the backend only forwards it to Gemini and streams the reply back. This keeps the backend fully stateless aside from the auth cookie, makes redeploys safe mid-conversation, and avoids needing any server-side storage (DB, Redis, or process memory) for chat history — consistent with the decision not to persist conversations at all.
